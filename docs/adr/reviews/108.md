### 검증 결과

Codex 리뷰 4 이슈 전부 VERIFIED (file:line evidence). Pre-design audit 4건 (containerVariants 사용 16 spec / 두 BASE_TAG_SPEC_MAP 차이 95vs108 / consumer 측 미소비 / orientation 4 spec) 으로 추가 확증.

### 핵심 발견

1. **CompositionSpec.containerVariants 가 이미 존재** — 16 spec 사용중, CSSGenerator 만 소비, Canvas/Panel 미소비. 진짜 SSOT 공백 = consumer 부재 (스키마 표현력 아님).
2. **두 BASE_TAG_SPEC_MAP 공존** — packages/specs:95 + builder:108 (15 alias). 이미 SSOT 위반 상태.
3. **자식 styling 은 nested[] selector 로 자연 표현 가능** — derivedContainerStyles 신규 함수 도입 불필요.

### 재설계 결정

ADR-108 r2 의 'derivedContainerStyles 함수 도입' 안 기각, r3 'containerVariants consumer 확장' 으로 전환:

- Spec 스키마 확장 0 (기존 containerVariants 사용)
- 공용 helper resolveContainerVariants(spec, props): {styles, nested[]} 신설
- selector mini-matcher (whitelist: > .react-aria-X 등) 신설
- Canvas/Panel 이 helper 호출 → 동일 데이터 소비
- Spec 'pure data' 원칙 유지
- Phase 5 follow-up ADR (CSSGenerator AST emit) 자체 소멸

### Phase 구성 (6+1)

- P0: Registry 통합 (95+15) + builder alias layer 분리
- P1: helper + selector mini-matcher + 16 spec selector audit
- P2: Canvas TextField PoC

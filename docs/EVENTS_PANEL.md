# Events Panel Smart Recommendations

## 계획
1. 현재 이벤트 등록/수정 플로우를 분석하고 병목을 정리한다.
2. 업계 트렌드(Webflow/Framer)를 조사해 핵심 패턴을 도출한다.
3. 추천 이벤트 노출 UX(빈 상태/리스트/디테일)와 노출 조건을 정의한다.
4. 컴포넌트 메타 + 이벤트 메타데이터로 추천 이벤트를 계산한다.
5. panel-contents에 추천 이벤트를 배치해 + 없이도 접근되도록 한다.
6. 단계별(P0/P1/P2) 개선안을 실행 계획으로 묶고 검증 기준을 정한다.

## 배경
현재 Events Panel은 헤더의 + 버튼만 제공한다. 사용자들은 기본 이벤트를 보기 위해 팝오버를 열어야 하며, 이는 접근성과 발견성을 낮춘다.

## 목표
- 선택 즉시 해당 컴포넌트의 기본/추천 이벤트를 panel-contents에 노출한다.
- 이벤트 추가 동작은 기존과 동일하게 유지한다(핸들러 생성 + 선택).
- 이벤트 레지스트리/지원 이벤트 정의는 변경하지 않는다.

## 비목표
- 신규 이벤트/액션 타입 추가.
- EventTypePicker 팝오버 UI 전면 재설계.
- 핸들러 디테일 편집 플로우 변경.

## 현재 이벤트 등록/수정 플로우 (as-is)
- 요소 선택 -> 헤더에 + 표시(지원 이벤트 1개면 단일 버튼).
- + 클릭 -> EventTypePicker 팝오버(검색/카테고리/추천).
- 이벤트 선택 -> 핸들러 생성 + 자동 선택.
- 블록 편집(When -> If -> Then/Else)로 수정.
- 액션 추가는 ActionPicker 오버레이에서 선택 후 편집.

## 단점
- 발견성 낮음: 기본 이벤트가 + 뒤에 숨겨짐.
- 단계가 많음: 팝오버를 열어야 시작 가능.
- 컨텍스트 전환: 팝오버/오버레이로 리스트 흐름이 끊김.
- 가이드 부족: 각 이벤트의 일반 사용 시나리오가 즉시 보이지 않음.
- 접근성 저하: 키보드 탐색 시작점이 헤더에 집중됨.
- 개인화 부재: 추천이 프로젝트/사용 패턴을 반영하지 못함.

## 업계 트렌드 리서치 (Webflow / Framer)

### Framer
- AI 생성: 텍스트 기반으로 레이아웃/컴포넌트/번역을 생성하는 AI 플로우 제공. [1]
- CMS 강화: 인라인 편집, SEO, 협업을 강조한 디자인 중심 CMS. [2]
- 폼/데이터 수집: 코드 없이 폼을 생성하고 데이터를 수집. [3]
- 내장 분석: 개인정보 친화적(쿠키 없는) 기본 분석 제공. [4]
- 로컬라이제이션: 언어/지역별 커스터마이징 지원. [5]

### Webflow
- AI 제공: Webflow AI 제품 페이지 운영. [6]
- CMS 중심 확장: 시각적 CMS 구축을 강조. [7]
- 로컬라이제이션: 번역/현지화 도구 페이지 제공. [8]
- 인터랙션/애니메이션: 시각적 애니메이션 제작을 강조. [9]

### 트렌드 요약 (2025)
- AI 기반 생성/보조: 빠른 초기 구성과 의도 기반 생성이 기본값.
- 템플릿/레시피화: “이벤트 + 액션”의 재사용 가능한 패턴 제공.
- 내장 분석/프라이버시: 간단한 인사이트 제공과 개인정보 친화 흐름.
- 다국어/로컬라이제이션: 글로벌 배포를 위한 기본 기능화.
- 인터랙션 중심 UI: 이벤트/상태 변화의 시각적 구성 강화.

## 2025 트렌드 기반 개선안 (전체 적용: P0+P1+P2)
1. 컨텍스트 추천 이벤트(quick add chips)로 즉시 발견성 확보.
2. 이벤트 선택 후 “추천 액션” 2~3개를 즉시 노출(Then 빠른 추가).
3. 레시피 템플릿(예: 폼 제출 -> 검증 -> 토스트)을 상단 추천으로 제공.
4. 의도 기반 생성(자연어 -> 이벤트+액션 초안) 옵션화.
5. 키보드 커맨드 팔레트(Ctrl+K)로 이벤트/액션 탐색.
6. 인라인 검증/설명: 호환성 배지, 누락 설정 경고, 미리보기 힌트.
7. 경량 시뮬레이션: mock payload로 액션 실행 테스트.
8. 개인화 추천: 프로젝트 내 사용 빈도 기반으로 추천 순서 재정렬.

## 적용 범위
- 전체 범위: P0 + P1 + P2.

## 단계별 실행
- P0: 추천 이벤트를 panel-contents에 노출 + 빈 상태 가이드 개선.
- P1: 추천 액션/레시피 템플릿 추가.
- P2: AI 의도 생성, 커맨드 팔레트, 개인화, 시뮬레이션.

## UX 상태
- 요소 미선택: 기존 빈 상태 유지.
- 요소 선택, 핸들러 없음:
  - 추천 이벤트 노출.
  - 빈 상태 메시지에 추천/ + 안내 추가.
- 요소 선택, 핸들러 있음(리스트 화면):
  - 추천 이벤트를 리스트 상단에 표시(남아있는 것만).
- 핸들러 디테일 화면:
  - 추천 이벤트 숨김(집중 편집).

## 데이터 소스
- 컴포넌트 지원 이벤트: `componentMeta.inspector.supportedEvents`.
- 구현 이벤트 필터: `isImplementedEventType`.
- 추천 이벤트 매핑: `COMPONENT_RECOMMENDED_EVENTS`.
- 우선순위: `EVENT_PRIORITY`.
- (선택) 개인화: 로컬 사용 빈도.

## 인터랙션
- 추천 이벤트 클릭 시 기존 add handler 경로 사용.
- 이미 등록된 이벤트는 추천 목록에서 제외.
- 추천이 없으면 섹션 자체를 숨김.

## 접근성
- 추천 버튼에 명확한 aria-label 제공.
- 키보드 포커스/활성화는 기존 버튼 규칙 준수.

## 스타일
- 기존 추천 chip 스타일 재사용.
- panel-contents 상단에 얇은 섹션으로 배치.

## 구현 노트
- 추천 이벤트 계산 로직은 공통 유틸로 분리.
- 라벨은 `EVENT_TYPE_LABELS` 재사용.
- 전역 스토어 변경 없이 패널 내부에 로직 배치.

## 검증
- Button: 클릭/키보드 이벤트 추천 노출 여부.
- TextField: 변경/키보드 이벤트 추천 노출 여부.
- 추천 이벤트 추가 -> 핸들러 생성 및 자동 선택 확인.
- 모든 이벤트 등록 시 추천 섹션 숨김 확인.

## 오픈 질문
- 핸들러 디테일 화면에서도 추천을 유지할지?
- 추천 액션/레시피를 어디에 노출할지?
- AI 의도 입력의 UX 위치(헤더 vs panel-contents) 결정.
